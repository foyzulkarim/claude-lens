OpenAI-compatible API

Dashboard Account Chat API

This server exposes a local OpenAI Chat Completions–compatible API, so any client that already speaks the OpenAI wire format (the official openai Python / Node SDKs, LangChain, llama.cpp clients, IDE plugins, etc.) can call your local Claude account by pointing its base_url at this server. Every call is proxied to Anthropic's /v1/messages with your local OAuth token — same quota as the Chat page, no separate API key required.

Multimodal: text + images

Both inputs ride on the standard OpenAI Chat Completions wire format. The user's content is an array of {type:"text"} and {type:"image_url"} blocks. The server handles two image source forms:

  • Inline data URL — data:image/png;base64,.... Use this for local files. Encode your bytes to base64 first.
  • Remote URL — any http:// or https:// URL Anthropic can fetch.

Allowed image MIME types: image/jpeg, image/png, image/gif, image/webp. Per-image size cap: ~5 MB (server-side limit before base64-inflation is 25 MB total request).

Python — local file

import base64
from openai import OpenAI

client = OpenAI(base_url="http://localhost:3456/v1", api_key="sk-not-required")

with open("diagram.png", "rb") as f:
    b64 = base64.b64encode(f.read()).decode()

resp = client.chat.completions.create(
    model="claude-haiku-4-5-20251001",
    messages=[{
        "role": "user",
        "content": [
            {"type": "image_url", "image_url": {"url": f"data:image/png;base64,{b64}"}},
            {"type": "text",      "text": "What's in this image?"},
        ],
    }],
    max_tokens=512,
)
print(resp.choices[0].message.content)

Node (openai SDK)

import fs from 'node:fs/promises';
import OpenAI from 'openai';

const client = new OpenAI({ baseURL: 'http://localhost:3456/v1', apiKey: 'sk-not-required' });
const b64 = (await fs.readFile('diagram.png')).toString('base64');

const resp = await client.chat.completions.create({
  model: 'claude-haiku-4-5-20251001',
  messages: [{
    role: 'user',
    content: [
      { type: 'image_url', image_url: { url: `data:image/png;base64,${b64}` } },
      { type: 'text',      text: 'What does this image show?' },
    ],
  }],
});
console.log(resp.choices[0].message.content);

Reference

What gets translated

Internally, the server converts your OpenAI request to Anthropic's /v1/messages shape and the response back. You don't need to know any of this — but here's the mapping in case something looks off:

messages[].role: "system"Pulled out of the array → Anthropic's separate system field (concatenated if multiple).
Multimodal image_urlBoth data:<mime>;base64,... URLs and remote https://… URLs are accepted. Mapped to Anthropic image blocks.
stopString or array → Anthropic stop_sequences.
finish_reasonend_turn/stop_sequence → stop, max_tokens → length, tool_use → tool_calls, refusal → content_filter.
usage.prompt_tokens= input_tokens + cache_read_input_tokens + cache_creation_input_tokens. prompt_tokens_details.cached_tokens reflects cache reads.
StreamingAnthropic SSE events are converted to OpenAI data: {chunk}\n\n chunks and a final data: [DONE].

Currently ignored fields

The following OpenAI fields are silently dropped (a warning is returned in the X-Claude-Lens-Warnings response header). The model still answers — just without these directives.

  • frequency_penalty, presence_penalty, logit_bias, seed — not exposed by Anthropic's API.
  • response_format (including {type: "json_object"}) — instruct the model in your prompt instead.
  • tools, tool_choice, parallel_tool_calls — function calling is not yet plumbed through this gateway.
  • n > 1, logprobs, top_logprobs, service_tier — not supported.

Models

The allow-list is the same as the Chat page: claude-opus-4-7, claude-sonnet-4-6, claude-haiku-4-5-20251001. To extend it, edit CHAT_MODELS in server.js.

Auth & quotas

Inference is billed against your local Claude account, so the same 5h / 7d unified quota you see on the Chat page applies here. Anthropic rate-limit headers (anthropic-ratelimit-unified-5h-utilization, retry-after, etc.) are forwarded on every response, so your client can decode 429s correctly.

The local API key (LOCAL_API_KEY) is purely a guard against accidental network exposure. Unset by default — perfect for localhost. If you forward port 3456 outside your machine, set one.
